## Project Overview

IMAG (Investment Management Game) is an Erasmus+ educational project providing teaching materials, engine documentation, and presentation slides for an investment management simulation (Cesim Invest). The project is a static site built with Quarto (R-based publishing system), hosted on GitHub Pages.

## Build Commands

Each sub-project is rendered separately from its own directory:

```bash
# Render the teaching book
quarto render teaching/

# Render the engine documentation
quarto render engine/

# Render the RevealJS slides
quarto render slides/
```

All outputs are written to `docs/` (i.e., `docs/teaching/`, `docs/engine/`, `docs/slides/`). There are no tests.

## CI/CD

| Workflow | Trigger | What it does |
|---|---|---|
| `quarto-build.yml` | push to `master`/`main` | Renders all three sub-projects, auto-commits `docs/` back to `master`. This is the active production pipeline. |
| `build-dev-preview.yml` | push to `dev` | Renders all three sub-projects with redirected `output-dir`, commits the result to `docs/dev/` on `master`. Preview is served at `sstoeckl.github.io/IMAG/dev/`. |
| `publish.yml`, `publish_gh.yml` | push to `main` | Legacy/broken — run `quarto render` at repo root where no `_quarto.yml` exists. |

The dev preview workflow temporarily patches each sub-project's `output-dir` via `sed` before rendering, then saves the output to `/tmp`, switches to `master`, and commits it to `docs/dev/`.

## Architecture

The repository has three independent Quarto sub-projects plus a static landing page:

```
teaching/   → Quarto book  → docs/teaching/
engine/     → Quarto website → docs/engine/
slides/     → Quarto RevealJS → docs/slides/
docs/index.html  → static landing page (hand-authored HTML)
```

**Shared includes** in `includes/` are injected into every sub-project's HTML output via each sub-project's `_quarto.yml`:
- `includes/head.html` — Tailwind CSS, Google Fonts, custom styles
- `includes/header.html` — sticky navigation bar
- `includes/footer.html` — copyright and Erasmus+ badge

Each sub-project's `_quarto.yml` references these with relative paths (`../includes/`).

## Content Structure

- `teaching/` — 13 chapter `.qmd` files + 3 appendix stubs + `references.bib`. Chapters follow the `chapters/XX-slug.qmd` naming convention.
- `engine/` — single-page documentation of the IMAG market model, asset universe, KPIs, and configuration.
- `slides/` — single `index.qmd` rendered as a RevealJS deck.
- `docs/index.html` — the project landing page; it is hand-authored HTML (not generated by Quarto) and includes the team accordion, video embed, and Erasmus+ branding.

## Key Conventions

- All source files are `.qmd` (Quarto Markdown). Do not create plain `.md` content files.
- Styling uses Tailwind CSS utility classes in the shared HTML includes and in `docs/index.html`. No separate CSS framework is added per sub-project.
- The `docs/` directory is checked into git as the build artifact (GitHub Pages serves from it).
- `_freeze/` directories inside each sub-project cache computed outputs; they are checked in so CI can render without re-executing R code.
